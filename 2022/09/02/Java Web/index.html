Connection

1,获取执行SQL的对象

  • 普通执行SQL对象

    Statement createStatement()

  • 预编译SQL的执行SQL对象：防止SQL注入

    PreparedStatement prepaerStatement(sql)

  • 执行存储过程的对象

    CallableStatement prepareCall(sql)

2,JDBC事务管理：Connection接口中定义的3个对应的方法

  • 开启事务

    setAutoCommit(boolean autoCommit): true为自动提交事务；false为手动提交事务，即为开启事务

  • 提交事务：commit()

  • 回滚事务：rollback()

Statement

  1. 执行SQL语句

    • int executeUpdate(slq):执行DML,DDL语句

      返回值：（1）DML语句影响的行数 （2）DDL语句执行后，执行成功也可能返回0

    • ResultSet executeQuery(sql):执行DQL语句

      返回值：ResultSet 结果集对象

ResultSet

  • 封装DQL查询的结果

PreparedStatement

  • 预编译SQL并执行，预防SQL注入

  • SQL注入：通过操作输入来修改事先定义好的SQL语句，用以达到执行代码对服务器进行攻击的方法:如下图所示

  • preparedStatement预防SQL注入（将敏感字符进行转义）

  • preparedStatement预编译功能开启：useServerPrepStmts=true(只需执行一次检查和编译)

数据库连接池

  • 数据库连接池是一个容器，负责分配，管理数据库连接（connection）

  • Druid连接池，功能强大

  • 获取当前用户路径
